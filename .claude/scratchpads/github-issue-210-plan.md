# Issue #210: Texture inpainting model architecture + UV texture dataset preparation

## Understanding
- Model 5 in Strata's ONNX pipeline — diffusion-based model that fills unobserved UV texture regions
- When Strata unwraps a 2D character painting onto a 3D mesh, back/sides have no texture data
- This model synthesizes plausible texture for those unseen UV regions
- **NOT a diffusion model** — the Strata Rust runtime (`texture_inpainting.rs`) expects a simple feedforward model with a single `"input"` tensor (5ch: RGBA + mask) → `"inpainted"` output (3 or 4ch RGBA)
- Resolution: 512×512 (MODEL_INPUT_SIZE in Rust runtime)

## ONNX Contract (from Strata runtime)
- **Input**: `"input"` — `[1, 5, 512, 512]` float32 (RGBA channels 0-3 + mask channel 4, where 1.0=observed, 0.0=unobserved)
- **Output**: `"inpainted"` — `[1, 4, 512, 512]` float32 (completed RGBA, 0-1 range)
- **Filename**: `texture_inpainting.onnx`
- **Budget**: ~15MB ONNX
- **Fallback**: ColorExtrapolation (BFS nearest-texel propagation)

## Approach
- U-Net architecture matching the inpainting model pattern (proven in codebase)
- 5-channel input (RGBA + observation mask), 4-channel RGBA output via sigmoid
- Same encoder-decoder with skip connections as `inpainting_model.py`
- Slightly deeper bottleneck to handle UV-space texture coherence (UV textures have different spatial structure than image-space)
- Keep within ~15MB ONNX budget

### Key difference from Model 4 (inpainting)
- Model 4 operates in **image space** (occluded body parts)
- Model 5 operates in **UV texture space** (unobserved texture regions from limited viewpoints)
- Model 5's mask semantics are **inverted** vs Model 4: mask=1 means OBSERVED (Rust runtime), not occluded
- Input is 5ch concatenated (not separate image + mask inputs) — Rust runtime builds a single 5ch tensor

### Why NOT diffusion
The issue title says "diffusion-based" but the Rust runtime (`texture_inpainting.rs`) expects a single forward pass model with one input tensor. A diffusion model would need iterative denoising which doesn't match the runtime. We'll use a deterministic U-Net — can always upgrade to a lightweight denoiser later if quality demands it.

## Dataset Layout
Pairs generated by `mesh/scripts/texture_projection_trainer.py` (already exists):
```
pairs_dir/
    char_001/
        partial_texture.png    (1024×1024 RGBA, from 3 views)
        complete_texture.png   (1024×1024 RGBA, from 24 views)
        inpainting_mask.png    (1024×1024 binary, 255=needs inpainting)
```

Dataset loader will:
- Load partial + complete + mask
- Resize to 512×512 for training (runtime operates at 512)
- Build 5ch input: RGBA from partial + inverted mask (1=observed, matching Rust convention)
- Target: complete_texture RGBA

## Risks & Edge Cases
- UV seams can cause artifacts at island boundaries
- Sparse UV layouts (lots of empty space) may confuse the model
- Alpha channel in UV textures has different semantics than image alpha
- Need to handle both 1024 source resolution and 512 training resolution
