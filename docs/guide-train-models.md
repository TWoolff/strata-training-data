# Guide: Train Strata's 4 Core ONNX Models

Step-by-step guide to train all 4 core models using existing bucket data and deploy them to the Strata desktop app.

## Prerequisites

### Hardware
- **A100 (recommended):** ~6-8h for all 4 models. Use `cloud_setup.sh lean` for automated setup.
- **Local GPU (RTX 3070+):** 8+ GB VRAM. ~12-24h for all models. Use default configs.
- **CPU:** Works but 10-50x slower. Not recommended.

### Software
```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r training/requirements.txt

# Verify GPU
python3 -c "import torch; print(f'CUDA: {torch.cuda.is_available()}')"
```

### Data Access
```bash
# Install rclone (macOS: brew install rclone, Linux: sudo apt install rclone)
# Configure with Hetzner credentials (see .env for keys):
rclone config  # Add remote "hetzner" with S3 provider, endpoint fsn1.your-objectstorage.com
```

## Quick Start: A100 Cloud

The fastest path. Spin up a Lambda Labs / Vast.ai A100, then:

```bash
export BUCKET_ACCESS_KEY='<from .env>'
export BUCKET_SECRET='<from .env>'
git clone https://github.com/TWoolff/strata-training-data.git && cd strata-training-data
./training/cloud_setup.sh lean              # Downloads data (~30min from tars)
rclone copy hetzner:strata-training-data/checkpoints/ ./checkpoints/ \
    --transfers 32 --fast-list -P           # Download previous checkpoints (optional)
./training/run_third.sh                     # Enrich + train all + upload + tar
# Destroy instance when done — everything is in the bucket
```

## Quick Start: Local GPU

```bash
# 1. Download training data
mkdir -p data_cloud/_tars
for ds in segmentation live2d humanrig anime_seg curated_diverse; do
    rclone copy hetzner:strata-training-data/tars/${ds}.tar data_cloud/_tars/ \
        --transfers 32 --fast-list -P
    tar xf data_cloud/_tars/${ds}.tar -C data_cloud/
    rm data_cloud/_tars/${ds}.tar
done

# 2. Train all models
./training/train_all.sh local

# 3. Export ONNX
# (train_all.sh does this automatically)
```

## Model 1: Segmentation (DeepLabV3+ Multi-Head)

5 output heads from a shared MobileNetV3-Large backbone:
- `segmentation` — 22-class body region logits [1, 22, 512, 512]
- `depth` — Marigold-distilled monocular depth [1, 1, 512, 512] (sigmoid)
- `normals` — Marigold-distilled surface normals [1, 3, 512, 512] (tanh)
- `confidence` — Foreground confidence [1, 1, 512, 512] (sigmoid)
- `encoder_features` — Backbone activations [1, 960, 64, 64] (for weight model)

### Training Data

| Dataset | Examples | Labels |
|---------|----------|--------|
| `segmentation/` | 1,598 | 22-class masks + depth + normals + joints |
| `live2d/` | 844 | 22-class masks + depth + normals |
| `humanrig/` | 11,434 | 22-class masks + depth + normals (partial) + joints + weights |
| `unirig/` | ~10,000 | 22-class masks + depth + normals (run 3) |
| `curated_diverse/` | 748 | fg/bg masks + depth + normals |
| `anime_seg/` | ~14,000 | fg/bg masks + joints |

Depth and normals labels are generated by Marigold LCM enrichment (`run_normals_enrich.py`). The training loop uses conditional losses — examples without depth/normals are still used for segmentation training.

### Train
```bash
# A100
python -m training.train_segmentation --config training/configs/segmentation_a100_lean.yaml

# Local
python -m training.train_segmentation --config training/configs/segmentation.yaml
```

### Key Metrics
- `val/miou` — Mean IoU across 22 classes. **Target: 0.55+**
- `val/depth_mae` — Depth mean absolute error. **Target: < 0.10**
- `val/normals_mae` — Normals mean absolute error. **Target: < 0.15**
- Early stopping: patience=20 on `val/miou`

### Best Score
Run 1: **0.5453 mIoU** (epoch 94/100, March 6 2026)

## Model 2: Joint Refinement (MobileNetV3 + Regression)

Predicts 2D positions of 20 skeleton joints from a character image.

### Training Data

| Dataset | Examples | Source |
|---------|----------|--------|
| `fbanimehq/` | ~101,630 | RTMPose estimates |
| `humanrig/` | 11,434 | Ground truth |
| `segmentation/` | 1,598 | Ground truth |
| `anime_seg/` | ~14,000 | RTMPose estimates |

### Train
```bash
python -m training.train_joints --config training/configs/joints_a100_lean.yaml
```

### Key Metrics
- `val/mean_offset_error` — Normalized offset error. **Target: < 0.002**
- `val/presence_accuracy` — Joint presence detection. **Target: > 0.97**
- Early stopping: patience=15 on `val/mean_offset_error`

### Best Score
Run 1: **0.001287** mean offset error (epoch 13, early stopped at 28)

## Model 3: Weight Prediction (Per-Vertex MLP)

Predicts skinning weights for 20 bones from per-vertex features. Optional encoder feature branch from Model 1's backbone provides visual context.

### Training Data

| Dataset | Examples | Notes |
|---------|----------|-------|
| `humanrig/` | 11,434 | Mixamo skeleton topology |
| `unirig/` | 14,950 | Diverse meshes (front views) |
| `segmentation/` | 1,598 | Mixamo renders |
| **Total** | **~27,982** | |

### Train
```bash
# Geometry-only (per-vertex MLP)
python -m training.train_weights --config training/configs/weights_a100_lean.yaml

# With encoder features (after segmentation model is trained):
# 1. Precompute encoder features
python -m training.data.precompute_encoder_features \
    --segmentation-checkpoint checkpoints/segmentation/best.pt \
    --data-dirs ./data_cloud/humanrig ./data_cloud/segmentation \
    --output-dir ./data_cloud/encoder_features \
    --only-missing

# 2. Train diffusion weight model
python -m training.train_diffusion_weights --config training/configs/diffusion_weights_a100_lean.yaml
```

### Key Metrics
- `val/mae` — Weight prediction MAE. **Target: < 0.05**
- `val/confidence_accuracy` — **Target: > 0.95**
- Early stopping: patience=20 on `val/mae`

### Best Score
Run 1: **0.0840 MAE** (54 examples only). Expecting major improvement with 27K examples.

## Model 4: Inpainting (U-Net)

Fills occluded body regions in character images (e.g., arm behind torso).

### Training Data
Generated on-the-fly from `fbanimehq/` by the occlusion pair generator:

```bash
python -m training.data.generate_occlusion_pairs \
    --source-dirs ./data_cloud/fbanimehq \
    --output-dir ./data_cloud/inpainting_pairs \
    --masks-per-image 3 --resolution 512 --max-images 15000
```

This creates ~45K occlusion pairs (~20 GB). Each pair has `masked.png`, `mask.png`, `target.png`.

### Train
```bash
python -m training.train_inpainting --config training/configs/inpainting_a100_lean.yaml
```

### Key Metrics
- `val/l1` — L1 reconstruction loss. **Target: < 0.05**
- `val/psnr` — Peak signal-to-noise ratio. **Target: > 25 dB**
- Early stopping: patience=15 on `val/l1`

### Best Score
Run 1: **BROKEN** (data loader bug, fixed in commit `011c3ba`). First real results expected in run 3.

## ONNX Export

After training, export all models to ONNX:

```bash
# Export all at once
python -m training.export_onnx --all --output-dir ./models/onnx/

# Or individually
python -m training.export_onnx --model segmentation \
    --checkpoint checkpoints/segmentation/best.pt \
    --output models/onnx/segmentation.onnx
```

### ONNX Contracts

| File | Input | Outputs |
|------|-------|---------|
| `segmentation.onnx` | `[1,3,512,512]` | segmentation `[1,22,512,512]`, depth `[1,1,512,512]`, normals `[1,3,512,512]`, confidence `[1,1,512,512]`, encoder_features `[1,960,64,64]` |
| `joint_refinement.onnx` | `[1,3,512,512]` | offsets `[40]`, confidence `[20]`, present `[20]` |
| `weight_prediction.onnx` | `[1,31,2048,1]` | weights `[1,20,2048,1]`, confidence `[1,1,2048,1]` |
| `inpainting.onnx` | image `[1,4,512,512]` + mask `[1,1,512,512]` | inpainted `[1,4,512,512]` |

Post-export validation runs automatically (onnxruntime shape + range checks).

## Deploy to Strata

```bash
cp models/onnx/*.onnx ../strata/src-tauri/models/
```

Strata loads models lazily from `models/` next to the executable (~55 MB total).
